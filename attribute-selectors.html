Attribute selectors

Case sensitivity flags

All attribute selectors, except for the "[attr]", follow the next syntax:

Syntax:
[attr operator value flag]

Where flag is optional and can be either i(or I) or s(or S). When the first is present, the attribute values will be comparated case-insensitively, and, when the second is present, those will be comparated case-sensitively.

This last fact only applies for attribute values with characters within the ASCII range, and case-sensitive matching via s flags is only possible in systems that preserve the original case, so it has less browser compatibility.

The Selectors

[attr]

Selects the elements with the "attr" attribute, whatever the value of the attribute.

[attr=value]

Selects the elements with an attribute "attr" whose value is exactly "value".

[attr~=value]

Selects the elements with an attribute "attr" whose value is a whitespace-separated list of words, one of which is exactly "value".

[attr|=value]

Selects the elements with an attribute "attr" whose value can be exactly "value" or can begin with "value" immediately followed by a hyphen (-).

It is often used for language subcode matches.

[attr^=value]

Selects the elements with an attribute "attr" whose value is prefixed (preceded) by "value".

[attr$=value]

Selects the elements with an attribute "attr" whose value is suffixed (followed) by "value".

[attr*=value]

Selects the elements with an attribute "attr" whose value contains at least one occurrence of "value" within the string.
